    ```python
    from pydantic import BaseModel
    from datetime import datetime
    from typing import Optional

    class CommentRecord(BaseModel):
        comment_id: int
        comment_type: str          # "review" | "issue"
        author: str                # GitHub login, or "[deleted]"
        body: str
        created_at: datetime
        url: str
        file_path: Optional[str] = None    # review comments only (per D-07)
        diff_hunk: Optional[str] = None    # review comments only (per D-07)
        reactions: dict[str, int] = {}     # non-zero counts only (per D-08)

    class PRRecord(BaseModel):
        number: int
        title: str
        body: Optional[str] = None         # PRs with no description (per pitfall 2)
        state: str                         # "open" | "closed"
        url: str

    class PRFile(BaseModel):
        pr: PRRecord
        comments: list[CommentRecord]
        extracted_at: datetime
    ```

    Per D-05: PRRecord is lean (number, title, body, state, url).
    Per D-06: CommentRecord is rich (comment_id, author, body, created_at, url, file_path, diff_hunk, reactions).
    Per D-07: file_path and diff_hunk are Optional, populated only for review comments.
    Per D-08: reactions is dict[str, int] with default empty dict.

    model_dump(mode="json") must serialize datetime as ISO 8601 (pydantic v2 does this automatically).
    model_validate() must reconstruct from the JSON-mode dict.
  </action>
