## Terminal Output Contract

This section replaces the web Spacing/Typography/Color sections for CLI-only phases.

### Output Channels

| Channel | Purpose | Click API |
|---------|---------|-----------|
| stdout | Summary lines, help text, command output | `click.echo()` |
| stderr | Per-item verbose detail, logging output | `logging.basicConfig(stream=sys.stderr)` |
| exit code 0 | Success | implicit / `sys.exit(0)` |
| exit code 1 | Runtime error (bad token, API failure, rate limit) | `raise click.ClickException(msg)` |
| exit code 2 | Usage error (bad flag, missing required option) | `raise click.UsageError(msg)` / Click built-in |

Source: RESEARCH.md — Pattern 3: ClickException for User-Facing Errors

### Color Usage (Terminal)

| Color | click.style fg= | Reserved For |
|-------|-----------------|-------------|
| Green | `"green"` | Success summary line at end of each command |
| Yellow | `"yellow"` | Warning — low-confidence flag in classify output |
| Red | implicit via ClickException | Errors only (Click prints "Error: ..." automatically) |
| Default (no color) | — | All per-item verbose output, help text |

Source: CONTEXT.md D-07 — colored output using `click.style()`; Click handles TTY detection automatically (no color when piped).

### Verbose Mode

| Mode | Behavior |
|------|---------|
| Silent (default) | No per-item output. Always print one summary line at the end. |
| Verbose (`-v` / `--verbose`) | `logging.basicConfig(level=logging.INFO, stream=sys.stderr)` — reuses existing `logger.info()` calls in extractor, classifier, generator. No additional CLI-level prints. |

Source: CONTEXT.md D-05, D-06; RESEARCH.md — Pattern 4

## Spacing Scale

Not applicable to terminal output. Click and the user's terminal control all whitespace in help text and echo output. No custom spacing tokens are declared for this phase.

Exception: Use a single blank line (`click.echo("")`) to visually separate the summary line from any preceding verbose log output.

---

## Typography

Not applicable to terminal output. System monospace font is used in all terminals. No font sizes or weights are declared for this phase.

Help text formatting rules (applied via Click docstring conventions):

| Element | Rule |
|---------|------|
| Command docstring | One-sentence imperative description. First line shown in group --help. |
| Option `help=` text | Lowercase, no trailing period. Example value included where helpful. |
| `\b` marker in docstring | Used before Examples block to disable Click's text rewrapping. |
| Example lines | Prefixed with two spaces inside `\b` block. |

Source: RESEARCH.md — Pattern 1 (code example), CLI-04 requirement

## Copywriting Contract

All copy is exact. Executor must use these strings verbatim.

### Command Help Text (--help output)

| Command | Docstring (first line) |
|---------|----------------------|
| `cli` group | `Extract, classify, and generate a knowledge base from GitHub PR discussions.` |
| `extract` | `Extract and cache PR comments from a GitHub repository.` |
| `classify` | `Classify cached PR comments into knowledge categories using Claude.` |
| `generate` | `Generate markdown knowledge base articles from classified comments.` |
| `run` | `Run the full extract → classify → generate pipeline in one command.` |

### Option Help Text

| Command | Option | Help text |
|---------|--------|-----------|
| `extract` | `--repo` | `GitHub repository in owner/name format. Example: pallets/click` |
| `extract` | `--state` | `Filter PRs by state. [default: all]` |
| `extract` | `--since` | `Only PRs updated on or after this ISO date. Example: 2024-01-01` |
| `extract` | `--until` | `Only PRs updated on or before this ISO date. Example: 2024-12-31` |
| `extract` | `--verbose / -v` | `Print per-PR detail during extraction.` |
| `classify` | `--verbose / -v` | `Print per-comment detail during classification.` |
| `generate` | `--verbose / -v` | `Print per-article detail during generation.` |
| `run` | `--repo` | `GitHub repository in owner/name format. Example: pallets/click` |
| `run` | `--verbose / -v` | `Print per-item detail for each pipeline step.` |

### Summary Lines (always printed, success path)

| Command | Summary copy | Color |
|---------|-------------|-------|
| `extract` | `Extracted {N} PRs, {M} comments cached.` | green |
| `classify` | `Classified {N} comments ({M} new, {K} cached).` | green |
| `generate` | `Generated {N} articles ({M} new, {K} skipped).` | green |
| `run` (each step) | Same as individual command summaries, printed in sequence. | green per step |

### Error Messages (actionable, no raw tracebacks)

| Error condition | Message |
|----------------|---------|
| Missing `GITHUB_TOKEN` | `Configuration error — missing required environment variable.\nHint: copy .env.example to .env and fill in GITHUB_TOKEN.` |
| Missing `ANTHROPIC_API_KEY` (classify) | `Configuration error — missing required environment variable.\nHint: copy .env.example to .env and fill in ANTHROPIC_API_KEY.` |
| Bad `--repo` format / repo not found | `Repository not found: {repo} — check the name or verify your token has repo read access.` |
| Rate limit exhausted | `{RateLimitExhaustedError message}` (pass through from extractor) |
| Generic extraction failure | `Extraction failed: {exc}\nHint: verify --repo is in owner/name format and your token has repo read access.` |
| Bad `--since` / `--until` format | `--{flag} must be an ISO date (e.g. 2024-01-01). Got: {value!r}` (UsageError, exit 2) |

Source: CONTEXT.md D-08, D-09; RESEARCH.md — Pattern 3, Error Wrapper Helper

### Run Command Mid-Pipeline Failure

| Scenario | Output |
|---------|--------|
| Extract succeeds, classify fails | Print green extract summary. Then: `Error: Pipeline failed at classify step: {reason}` (ClickException, exit 1). |
| Extract + classify succeed, generate fails | Print green extract + classify summaries. Then: `Error: Pipeline failed at generate step: {reason}` (ClickException, exit 1). |

Source: CONTEXT.md D-10; RESEARCH.md — Open Question 2

### Example Blocks (shown in --help via \b marker)

| Command | Examples block |
|---------|---------------|
| `extract` | `github-pr-kb extract --repo pallets/click`\n`github-pr-kb extract --repo pallets/click --state closed --since 2024-01-01` |
| `classify` | `github-pr-kb classify`\n`github-pr-kb classify --verbose` |
| `generate` | `github-pr-kb generate`\n`github-pr-kb generate --verbose` |
| `run` | `github-pr-kb run --repo pallets/click` |

Source: CLI-04 requirement; RESEARCH.md — Pattern 1

### Empty State Copy

| Command | Condition | Output |
|---------|-----------|--------|
| `extract` | Zero PRs match filters | `Extracted 0 PRs, 0 comments cached.` (green, not an error) |
| `classify` | No unclassified comments in cache | `Classified 0 comments (0 new, 0 cached).` (green, not an error) |
| `generate` | No classified comments to generate from | `Generated 0 articles (0 new, 0 skipped).` (green, not an error) |

### Destructive Actions

None in this phase. No delete, overwrite, or irreversible operations are exposed via the CLI. Cached data is preserved across re-runs (D-10).
