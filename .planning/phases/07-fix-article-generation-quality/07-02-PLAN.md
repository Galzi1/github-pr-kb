**Rewrite _build_article (D-01, D-02, D-03, D-04, D-15):**

Keep the frontmatter and title-heading structure, but change the method to return `str | None`.

Core behavior:
1. Select section headings from `_CATEGORY_SECTIONS`.
2. Build a user prompt containing:
   - PR title
   - category
   - raw comment body (trimmed to a safe max, e.g. 10k chars)
   - the exact section headings
   - an explicit reminder that unsupported sections must say `Not stated in the source comment.`
3. Call `self._client.messages.create(...)`.
4. On `anthropic.APIError`, record failure and return `None`.
5. Extract text via `_extract_synthesized_body(response)`. If `None`, record failure with a clear reason such as `EmptySynthesis` or `UnexpectedResponseShape`, then return `None`.
6. If `_looks_like_source_echo(comment.body, synthesized_body)` is true, record failure with a reason such as `SourceEchoRejected`, then return `None`.
7. Build article content from:
   - frontmatter
   - `# {classified.summary}`
   - synthesized body
   - optional fenced `diff_hunk`

Important constraints:
- Raw `comment.body` may appear in the prompt only, never in the article body.
- Treat bad-but-non-exceptional model output as a synthesis failure, same no-write path as API exceptions.
- Preserve provenance through frontmatter (`pr_url`, `comment_url`, etc.), not by embedding source text.

**Update _write_article to handle None from _build_article (D-15):**

If `_build_article(...)` returns `None`, return `None` immediately without creating directories, writing files, or mutating slug caches.

**Add confidence filtering in _process_classified_file (D-14):**

**Replace destructive _reset_kb with transactional regenerate support (D-16, risk R4):**

Do NOT clear the live manifest or delete live category directories up front.

Instead, add a staged regenerate helper (name it `_generate_all_transactionally` or equivalent) with this behavior:

1. Create a sibling staging directory with `tempfile.mkdtemp(...)` under `self._kb_dir.parent`.
2. Temporarily point generation state at the staging directory:
   - `_kb_dir = stage_dir`
   - `_manifest = {}`
   - `_category_slugs = {}`
3. Run the normal generation pass into the staging directory, then save manifest and generate index there.
4. If generation aborts with an unexpected exception before staging finishes, delete the staging dir and leave the live KB untouched.
5. If staging succeeds, swap directories:
   - rename existing live KB dir to a backup sibling if it exists
   - rename the staged KB dir into the live path
   - if swap fails, restore the backup
   - only delete the backup after the live path has been replaced successfully
6. Reload `self._manifest` from the live KB path after the swap so in-memory state matches disk.

This plan intentionally treats D-16 as an atomic replacement of KB state, not a pre-emptive wipe.

**Refactor generate_all to use a shared generation pass and expose regenerate semantics clearly:**

1. Extract the current loop/save/index logic into a helper that generates into the current `self._kb_dir`.
2. Change the public method to:
```python
def generate_all(self, regenerate: bool = False) -> GenerateResult:
```
3. Reset `_written`, `_skipped`, `_filtered`, `_failed`, and `_category_slugs` at the start of every run.
4. When `regenerate=False`, run the normal generation pass.
5. When `regenerate=True`, run the transactional staged pass.
6. Return:
```python
GenerateResult(
    written=self._written,
    skipped=self._skipped,
    filtered=self._filtered,
    failed=self._failed,
)
```
7. Update the `generate_all()` docstring to state that threshold/model/prompt changes only affect already-generated articles when `regenerate=True`.
