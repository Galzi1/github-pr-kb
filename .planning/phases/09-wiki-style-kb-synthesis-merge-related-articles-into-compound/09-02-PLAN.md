<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude API -> topic page body | LLM-generated markdown could contain arbitrary links or content |
| Topic page body -> cross-ref links | Links could point to non-existent pages |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | Tampering | Claude-generated cross-ref links | mitigate | _strip_broken_links validates all relative links against known topic slugs; broken links converted to plain text (D-11) |
| T-09-05 | Information Disclosure | Topic synthesis prompt | accept | Source article bodies sent to Claude API - same trust boundary as existing per-comment synthesis; no new PII exposure |
| T-09-06 | Denial of Service | Large topic groups | mitigate | Source comment bodies truncated to 10,000 chars each (matching existing _build_synthesis_prompt pattern); max_tokens caps response |
</threat_model>
