  <action>
Create `.github/workflows/ci.yml` with the following configuration:

**Triggers:**
- `push` to any branch
- `pull_request` targeting `main`

**Job: `test`**
- runs-on: `ubuntu-latest`
- Strategy matrix: Python 3.11 (matches `requires-python = ">=3.11"` in pyproject.toml)
- Steps:
  1. `actions/checkout@v4`
  2. Install uv using `astral-sh/setup-uv@v5` (this also handles Python installation via uv)
  3. `uv sync` to install all dependencies including dev group
  4. Lint: `uv run ruff check src/ tests/` — fail fast before tests
  5. Unit + integration tests: `uv run python -m pytest tests/ -v` with env vars:
     - `GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}` — GitHub Actions provides this automatically, satisfies the integration test skip guard (settings.github_token != _DUMMY_TOKEN)
     - `RUN_INTEGRATION_TESTS: "1"` — enables the integration test marker

IMPORTANT notes:
- Use `uv run python -m pytest` (not bare `pytest`) to ensure the correct interpreter. In CI with uv-managed venv this is reliable (the local Windows issue from CLAUDE.md does not apply to Linux CI).
- The `secrets.GITHUB_TOKEN` is automatically available in GitHub Actions — no manual secret setup required.
- Integration tests hit the real GitHub API (read-only, public repos) which works fine with the default GITHUB_TOKEN.

**Job: `lint` (separate job for fast feedback)**
Actually, keep it as a single job to avoid duplicating the uv setup. Run ruff before pytest so lint failures are caught first.
